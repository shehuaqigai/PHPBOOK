GNU Autoconf 是一个在Bourne shell下制作供编译、安装和打包软件的配置脚本的工具[1]。 Autoconf并不受程序语言限制，常用于C、C++ 、Erlang和Objective-C。 配置脚本控制了一个软件包在特定系统上的安装。 在进行一系列测试后，配置脚本从模板中生成makefile与头文件进而调整软件包，使之适应某一种系统。 Autoconf与Automake、Libtool等软件组成了GNU构建系统[2]。 Autoconf 由 戴维·麦肯思于1991年夏天编写用于支持他在自由软件基金会的编程工作[3]。 此后，Autoconf包含了多人编写的改进代码并成为了使用最广泛的自由编译配置软件。 可以通过yum install autoconf来安装
